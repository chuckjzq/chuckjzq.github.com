spark入门学习-RDD基本操作

Transformation就是从之前的RDD构建一个新的RDD， 像map() 和 filter().
After Spark 2.0, RDDs are replaced by Dataset, which is strongly-typed like an RDD, but with richer optimizations under the hood. The RDD interface is still supported, and you can get a more complete reference at the RDD programming guide. However, we highly recommend you to switch to use Dataset, which has better performance than RDD.

逐元素Transformation

  • map()：map()接受函数，把函数应用到RDD的每一个元素，放回新的RDD。
  • filter(): filter()接收函数，返回只包含满足filter()函数的元素的新RDD。
  • flatMap(): flatMap()对每个输入元素，输出多个输出元素，flat压扁的意思，将RDD中的元素压扁后放回一个新的RDD，也就是变成一维的数组。
  • var定义变量，val定义常量，常量定义以后就不可以修改了。

集合运算

RDDs支持集合运算，例如去重、并集、交集、差集。

Action

在RDD上计算出来一个结果。
把结果返回给driver program或保存在文件系统，count(),save

  • reduce(): 接受一个函数，作用在RDD两个类型相同的元素上，返回新元素。
  • 可以实现，RDD中元素的累加，技术，和其他类型的聚集操作。
  • collect(): 遍历整个RDD，向driver program返回RDD的内容，需要单机内存能容纳下。大数据的时候，使用saveAsTextFile() action等。
  • take(n): 返回RDD的n个元素（同时尝试访问最少的partitions），返回结果是无序的，测试使用。
  • top(n): 排序后取回前几个。
  • foreach(): 计算RDD中的每个元素，但不放回到本地。可以配合println()友好的打印数据。

RDDs的特性

  • RDDs的血统关系图；
  • 延迟计算（Lazy Evaluation）: Spark对RDDs的计算是，他们第一次使用action操作的时候。这种方式在处理大数据的时候特别有用，可以减少数据的传输。Spark内部记录metadata 表名 transformations操作已经被响应了。加载数据也是延迟计算，数据只有在必要的时候，才会被加载进去。
  • RDD.persist(): 默认每次在RDDs上执行action操作的时候，Spark都会重新计算RDDs，如果想重复利用计算RDD，可以使用RDD.persist()，还可以设定缓存级别内存中还是磁盘中。
  • unpersist()从缓存中移除。

key-value对RDDs

  • 创建KeyValue对RDDs：
    • 使用map()函数返回key/value对。
    • keyvalue对的 操作。
  • combineByKey():
    • (createCombiner, mergeValue, mergeCombiner, partitioner)
    • 最常用的基于key的聚合函数，返回的类型可以和输入的类型不一样。
    • 许多基于key的函数都用到了他，像groupByKey().
    • 遍历分区Partition中的s元素，元素中的key，要么之前见过的，要么没见过的。
    • 如果是新元素，那么使用createCombiner()函数
    • 如果是这个Partition中已经存在的key，那么使用mergeValue().
    • 合计每个分区partition的时候 ，使用功能mergeCombiners().
    • 其实就是先每个分区合并自己的value，然后再合并分区。
