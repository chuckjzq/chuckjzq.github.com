记录几个我觉得非常有用的技巧，其他相关的教程可以参考这篇文章

使用Ulysses管理我的md文件使用Mweb来管理md文件

由于Ulysses对于代码非常不友好，在格式和缩进上都不支持，所以弃用。
并且发现Mweb用起来更爽。
使用Ulysses添加外部文件夹Mweb引入文件夹的功能很好的管理我们的文章，并且可以边写边在server网页上预览。

MWeb 是有两种模式的：外部模式和文档库模式。MWeb 为了满足所有 Markdown 使用需求，设计了两种模式！外部模式用于打开和编辑所有本地 Markdown 文档。另外为了方便用 Markdown 记笔记，MWeb 设计了文档库模式，文档库中的文档也支持一键生成静态博客，以便于分享。关于两个模式的情况，还是建议您去看一下官网的视频和帮助。
所以使用外部模式来管理md文件。

插入图片

在Mac上我Ulysses来写markdown，并且Ulysses有一个最大的优点就是可以关联一个外部文件夹，当我们创建了一个新的md文件时Ulysses上也就显示出来了。而且可以直接拖图片进来,但是链接只需要保留 【xxxx/图片名.jpg】一个目录名，可以参考这篇博客。
插入图片的时候要求还是很严格的，首先“hexo new “md-name””md的文件名需要是英文的，因为存放图片的文件夹需要是英文的，而且不能先是中文再改成英文。我们可以先让md文件名为英文，然后把title改为英文的，这样文章的标题就是可以是中文的了。

新建页面

hexo new page "page name"

草稿功能

首先在source 的目录下新建一个_drafts文件然后就可以用下面这个命令新建草稿。当想要发布的时候使用publish发布。
hexo new draft "test draft"
发布
hexo publish "test draft"
程序会把要发布的文件移到_posts文件中去。

本地调试

可以边写md文件边查看效果。只要保持server一直运行即可，不需要反复的hexo generate，等到满意的时候再deploy。

标题等级

使用三级标题正好比文章的标题大一点，但是如果文章很长也可以用一级标题开始写。
