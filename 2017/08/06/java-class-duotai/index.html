面向对象三大特性 之多态

多态概念

对象的多种形态，分为两种。

  1. 引用多态
    • 父类的引用可以指向本类的对象
    • 父类的引用可以指向子类的对象

前面的Animal就相当于变量的类型，后面的是新建Animal()的对象。所以Animal的引用可以指向Dog()对象。

  1. 方法的多态
    • 创建本类对象时，调用的方法为本类的方法
    • 创建子类的对象时，调用的方法为子类重写的方法或者继承的方法。

注意：1）这里的引用和对象的概念。2）父类里面没有的方法，父类引用无法调用。

同样都是父类的引用，可以指向不同对象的方法。

@override
这个用于重写父类方法用的
用这个注解可以避免方法名跟参数写错

引用类型转换

  1. 向上类型转换（隐式/自动类型转换），从小类型到大类型。
  2. 向下类型转换（强制类型转换），从大类型到小类型。
  3. 注：向上转换无风险，向下类型转换有风险（例如水壶里的水倒到杯子里）。
  4. instanceof运算符，来解决引用对象的类型，避免引用类型转换的安全性问题。

抽象类

  1. 定义

    • 抽象类前面用abstract关键词修饰，则这个类为抽象类。

  2. 应用场景

    • 在某些情况下，某个父类只知道其子类应该包含哪些方法，但不知道具体如何实现，即用抽象方法来约束子类必须包含哪些方法。
    • 从多个具有相同特征的类中抽象出抽象类，以这个抽象类做为模板，避免子类设计的随意性。

  3. 作用

    • 限制规定子类必须实现某些方法，但不关注实现细节。

  4. 使用规则

    • abstract定义抽象类
    • abstract定义抽象方法，只需要声明，不需要实现。（这里的声明和c++中的有点像，不需要实现）
    • 包含抽象方法的类是抽象类。
    • 抽象类中可以包含普通方法，也可以没有抽象方法。
    • 抽象类不能直接创建，可以定义引用变量。（意思是抽象类不能实例化，但是可以定义引用变量指向子类的对象）

接口

接口是一个规范。

  1. 类是一种具体实现体，而接口定义了某一批类所遵守的规范，接口不必关心这些类的内部数据，也不必关系这些类的里面方法内部的具体实现，它只规定这些类必须提供某些方法。
  2. 接口的定义
    • 和类不同，定义接口不再使用class关键字，而是使用interface关键字。
    • 常量：接口中的属性是常量，所以要加public static final修饰符，如果不加系统会自动加。
    • 方法：接口中的方法只能是抽象方法（这个和抽象类不一样，抽象类中可以是普通方法），规定要public abstract修饰，如果不加会自动加上。
  3. 使用接口
    • 一个类可以实现多个接口，使用implements关键字。
    • java中只能继承一个父类，是不够灵活的，可以利用接口弥补这个不足。
    • extends 父类 implements 接口1，接口2.顺序不能变。
    • 使用接口的时候，还经常和匿名内部类配合使用。
  4. 在给接口取名的时候可以在名字前面加个I，以区分和类名不一样。

interface前面可以加abstract关键字，如果没有系统会默认加，修饰符必须是public。可以继承多个接口。
