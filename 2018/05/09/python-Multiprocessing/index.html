python 中的多进程

多进程 Multiprocessing 和多线程 threading 类似, 他们都是在 python 中用来并行运算的. 不过既然有了 threading, 为什么 Python 还要出一个 multiprocessing 呢? 原因很简单, 就是用来弥补 threading 的一些劣势。

总之：多进程是利用cpu的多个核，比多线程高效（python）

使用步骤

导入标准模块 –> 定义被进程调用的函数 –> 创建进程 –> 启动&连接进程

进程池 Pool（高级而简单的方法，建议使用这个而不是Process）

进程池就是我们将所要运行的东西，放到池子里，Python会自行解决多进程的问题.

Pool和之前的Process的不同点是丢向Pool的函数有返回值，而Process的没有返回值。

使用流程

创建一个进程池 –> 用map()获取结果

Pool除了map()外，还有可以返回结果的方式，那就是apply_async().

共享内存

在多进行处理的时候，往往变量是相互独立的。那么想对相同的一个变量进行计算的时候该如何做呢？那就需要共享内存了。
我们可以通过使用Value数据存储在一个共享的内存表中。

进程锁 Lock

如果构造了共享内存的变量，那么多个进程都会争先恐后的对这个变量进行处理，那么如何防止争抢，那么就需要一把锁，把门锁住，一次只能进来一个人。
